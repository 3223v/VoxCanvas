# 设计文档：能力状态

## 一、概述

两层 LLM 架构：第一层负责意图理解和任务拆解，第二层负责样式细化。

***

## 二、计划支持的指令能力

项目计划支持以下四大类指令能力：

### 2.1 CREATE（创建）

| 形状类型        | 描述    | 预期能力            |
| ----------- | ----- | --------------- |
| `rect`      | 矩形    | 创建各种面板、容器、流程步骤  |
| `circle`    | 圆形    | 创建节点、状态、开始/结束标识 |
| `ellipse`   | 椭圆    | 创建数据库、存储图标      |
| `diamond`   | 菱形    | 创建判断节点、条件分支     |
| `line`      | 直线/折线 | 自由画笔、连线、多边形     |
| `dashed`    | 虚线    | 辅助线、透视线、草稿线     |
| `arrow`     | 箭头线   | 方向指示、流程连接       |
| `arc-arrow` | 弧线箭头  | 弯曲流程、回环连接       |
| `text`      | 文字    | 标签、标题、说明文字      |

### 2.2 MODIFY（修改）

| 修改类型 | 描述     | 预期能力         |
| ---- | ------ | ------------ |
| 位置调整 | 移动对象   | 支持绝对坐标和相对偏移  |
| 尺寸调整 | 缩放对象   | 调整宽度和高度      |
| 样式修改 | 改变外观   | 颜色、填充、描边粗细   |
| 模糊意图 | 自然语言描述 | "更醒目"、"改成蓝色" |

### 2.3 DELETE（删除）

| 功能   | 描述     | 预期能力               |
| ---- | ------ | ------------------ |
| 单点删除 | 删除单个对象 | 支持通过 label/位置/类型定位 |
| 级联删除 | 删除关联连线 | 自动清理 connected 箭头  |

### 2.4 CONNECT（连接）

| 连线类型 | 描述   | 预期能力      |
| ---- | ---- | --------- |
| 直线箭头 | 默认连接 | 两个对象中心点之间 |
| 虚线连接 | 弱关系  | 可选路径、辅助连接 |
| 弧线连接 | 弯曲流程 | 避开障碍物     |
| 双向箭头 | 双向关系 | 双向流程      |

### 4.2 已知限制

#### 4.2.1 绘图性能

- **问题**：画布对象过多时重绘性能下降
- **原因**：`RoughCanvas.redraw()` 使用全量重绘策略
- **影响**：大量对象时响应变慢

#### 4.2.2 LLM 调用成本

- **问题**：复杂指令需要多次 LLM 调用
- **示例**："画 A、B、箭头连接" 需要 4 次调用（1 次 task-generate + 3 次子工作流）
- **原因**：双层 LLM 架构设计

#### 4.2.3 文本渲染

- **问题**：文字没有手绘效果
- **原因**：Rough.js 不支持文字，使用 Canvas 原生 API
- **影响**：文字与手绘风格不一致

#### 4.2.4 撤销功能

- **问题**：仅支持前端本地撤销
- **原因**：数据库级 undo 功能未实现
- **影响**：刷新页面后撤销历史丢失

### 4.3 最大的限制

最重要的 task-generate 工作流，是系统的核心，根据用户的输入，生成对应的任务。

因为它只调用一次LLM，能力有限，难以处理超长上下文，极复杂指令，复杂画面的绘制难以控制。

绘制复杂画面时，精细调整对象的位置、尺寸、样式，对它来说并不容易。


所以，对于更复杂的指令，比如连接多个对象，它需要输出大量任务来实现，控制精度下降，子工作流任务繁多。导致绘制时间增加，影响用户体验。
